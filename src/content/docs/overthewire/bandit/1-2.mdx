> Goal: read the password from a file whose entire name is a single dash.

## Approach

```bash frame="code" title="bandit1@bandit"
bandit1@bandit:~$ ls
-
bandit1@bandit:~$ cat ./-
The password you are looking for is: <password>
```

## Why it works

cat reads its filename as an argument, and a token that starts with a dash is parsed as an option before it is ever treated as a path. Giving the name as ./- (or feeding the file through stdin) forces it to be read as a file, not a flag.
