> Goal: clone the level's git repository and read the password it stores.

## Approach

```bash frame="code" title="bandit27@bandit"
bandit27@bandit:~$ cd "$(mktemp -d)"
bandit27@bandit:/tmp/tmp.NxwmnLaKZF$ git clone ssh://bandit27-git@localhost:2220/home/bandit27-git/repo.git
Cloning into 'repo'...
bandit27@bandit:/tmp/tmp.NxwmnLaKZF$ cat repo/README
The password to the next level is: <password>
```

## Why it works

A git remote you can authenticate to is just a readable data source. Cloning copies the working tree (and the full history) to a directory you own, so a secret committed into the README comes along with it. This is the setup for the next three levels, which all mine a git repo.
